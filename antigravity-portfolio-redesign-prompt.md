Build this as a production-ready **developer portfolio** for Krishnapal Sendhav, a software developer focused on Flutter, backend engineering, AI/LLM integration, product building, and scalable apps. The portfolio should communicate strong engineering depth, modern product taste, and credible real-world execution. ### Design system requirements

Use these design rules exactly unless a better implementation still clearly honors the same visual DNA:
- Background/surfaces must be dark and layered, with glass-inspired panels and thin border contrast.[cite:41]
- Primary accent: #00F0FF for key emphasis, CTA signals, active states, highlights.[cite:41]
- Tertiary accent: #0A45FF only for selective support moments, not everywhere.[cite:41]
- Neutral/contrast moments: white and soft gray text hierarchy.[cite:41]
- Typography pairing:
  - Display: Cinzel for key hero/display headings only.[cite:41]
  - Body/UI: Manrope for paragraphs, labels, buttons, metadata, nav, cards.[cite:41]
- Spacing system:
  - Base rhythm: 16px
  - Scale should follow 16, 24, 32, 40, 48 where appropriate
  - Avoid random spacing values.[cite:41]
- Labels can use uppercase tracked styling where appropriate.[cite:41]
- Radius should remain controlled and aligned to the existing system; do not introduce inconsistent corner styles.[cite:41]
- Border treatments should feel premium and subtle, including gradient shell or hairline edge treatments where useful.[cite:41]
- Motion should stay moderate and intentional: roughly 150ms micro interactions, up to 500ms for section reveals; use ease / cubic-bezier(0.4, 0, 0.2, 1); no flashy motion overload.[cite:41]
- Preserve the “retro-futurist technical calm” mood across every section.[cite:41]
