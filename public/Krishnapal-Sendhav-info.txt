CONTENT UPLOAD & PROCESSING PIPELINE

I proposed, designed, and implemented a complete content upload and processing pipeline for educational videos. This system converts uploaded videos into multiple quality formats, encrypts media content for secure delivery, and uploads content to the server in an optimized manner.

Before this pipeline, institutes had to wait several hours after uploading lectures before they became available to students. After implementation, lectures became available instantly. This change also resulted in more than a 35 percent reduction in server and processing costs.


LIVE CLASS CHAT & REAL-TIME SYSTEMS

I implemented live class chat and real-time session management using Firebase Realtime Database. Prior to this, the platform relied on a server-hosted WebSocket solution that cost approximately 25,000 to 30,000 INR per month.

After migrating to Firebase Realtime Database, latency was reduced, scalability improved, and monthly infrastructure costs dropped to approximately 2,000 to 2,500 INR. This migration also simplified maintenance and improved overall system reliability.


WEBRTC VIDEO ROOM – LIVE SESSION MODULE

I recently developed a WebRTC-based video room functionality inside the Live Session module of the ClassIO student app. This feature enables real-time video communication for live interactions between students and educators.

The implementation uses a Star Topology (serverless architecture) with Firebase as the signaling mechanism, reducing infrastructure cost and latency significantly. Built entirely in Flutter, handling WebRTC peer connection management, media stream handling, signaling, and full session lifecycle.

Currently live in production with real users, supporting 10–15 participants simultaneously, with ongoing in-house testing to scale further.

Impact: Previously students were restricted to text chat only. Now, if faculty allows, students can interact via direct video for doubts and discussions. This transformed the platform from one-way broadcasting to two-way interactive conversations, helping faculty focus on students who need extra attention and significantly improving trust and engagement between students and educators.


AI DOUBT RESOLUTION SYSTEM
